# ROADMAP

План переноса веб-резюме из `legacy/` в Astro и публикации на GitHub Pages.
README.md остаётся пустым до завершения проекта.

## Стек и решения

- Astro + bun, статическая сборка (SSG).
- Контент резюме вынесен в JSON, валидируется zod-схемой через Astro data
  collection. Коммит нового JSON обновляет страницу без правки компонентов.
- i18n: отдельные маршруты `/` (ru) и `/en/` (en), конфигурация Astro i18n.
- Тема: `data-theme` на `<html>`, CSS-переменные, светлая/тёмная. Печать
  всегда светлая.
- Печать Ctrl+P: веб-страница как лист A4, `@media print` форсирует светлую
  тему и печатает только резюме.
- Доступность как сквозное требование, без отдельных контрастных тем.
- Деплой: GitHub Actions (`bun install` + `astro build` -> GitHub Pages),
  собранный `dist` не коммитится.

## Этапы

### 0. Инструментарий: форматтер, линтер, git-хуки

- Prettier (с `prettier-plugin-astro`) как форматтер.
- ESLint (с `eslint-plugin-astro`): неиспользуемые импорты, a11y-правила
  astro, ошибки в скриптах.
- husky + lint-staged.
- `pre-commit`: форматирование и линт staged-файлов.
- `pre-push`: полный `astro check` всего проекта.

### 1. Фундамент Astro и i18n

- Настроить `astro.config.mjs`: i18n (defaultLocale `ru`, locales `ru`/`en`),
  site URL для Pages.
- Проверить strict tsconfig, завести `src/data/` и `src/components/`.
- Скелет страницы-обёртки резюме (пока пустой контент).

### 2. JSON-контент и zod-схема

- Описать структуру данных резюме (контакты, опыт, навыки, образование и т.д.).
- zod-схема в data collection, TS-тип из неё.
- Наполнить `cv.ru.json` и `cv.en.json` из `legacy/index.html`.

### 3. Вёрстка резюме как лист A4

- Разнести вёрстку `legacy/index.html` по `.astro`-компонентам.
- Перенести layout и `cv-layout.css` (лист A4 на веб-странице).
- Семантическая разметка, логичный порядок блоков в исходном HTML.

### 4. Тема (light/dark)

- CSS-переменные из `legacy/css/variables.css`.
- `data-theme` на `<html>`, переключатель, сохранение в localStorage.
- Дефолт по `prefers-color-scheme`.

### 5. Печать

- Перенести `legacy/css/print.css`.
- `@media print`: форсировать светлую тему, скрыть UI (переключатели
  темы/языка), `@page { size: A4 }`, печатать только резюме.

### 6. Переключение языков

- Маршруты `/` и `/en/` на основе i18n-конфига.
- Переключатель языка, локализованные `<meta>` и JSON-LD.

### 7. JS-поведение

- Перенос `legacy/js/script.js`: переключение темы, копирование текста,
  вставка JSON-LD `schema.org/Person`.
- Адаптировать под Astro (islands / inline-скрипты по необходимости).

### 8. Доступность

- Контраст по WCAG AA, видимый фокус, alt-тексты, `prefers-contrast`/
  `forced-colors`, работоспособность без картинок.
- Проверка порядка блоков скринридером/инструментами.

### 9. Стили (общее)

- Перенос `reset.css`, `styles.css`, общий внешний вид и типографика.

### 10. Деплой на GitHub Pages

- Workflow `.github/workflows/`: install, build, deploy-pages.
- Настроить base path, если репозиторий не корневой.
- Проверить сборку и публикацию.

### 11. README

- Описать проект, стек, команды, деплой.
